4 为什么聚合签名验证更快

具体公式推导没有看懂，总之可以分成两种运算，其中可合并ML后运算FE。

对单个配对 来说：

  1. Miller loop (ML)：算出 ，但它还没在目标子群 ；

  2. Final exp (FE)：把结果提到指数 ，得到

所以配对的“真正结果” = ML 输出 + 一次 FE。

对于逐个验证： 需要做 2n 次 ML + 2n 次 FE。

而聚合签名： 左侧做 1次ML + 1次FE，右侧可合并ML后做FE，即 n 次 ML + 1次 FE。 总共 n+1 次ML + 2次 FE。
